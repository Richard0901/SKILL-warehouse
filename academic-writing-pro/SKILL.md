## 模块一：论文结构写作

### 沙漏模型

论文整体呈现"倒三角-矩形-正三角"结构：
- **Introduction（倒三角）**：宏观背景 → 具体研究问题
- **正文（矩形）**：严谨、连贯的论证
- **Conclusion（正三角）**：具体发现 → 宏观意义

### Introduction 三阶段结构

1. **广阔背景**：领域现状 + 研究必要性（重要性 + 研究空白）
2. **具体领域**（5W1H框架）：
   - What：研究主题
   - Why：研究动机
   - Where：研究场域
   - Who：研究对象
   - When：时间范围
   - How：研究方法
3. **Thesis Statement**：核心主张，清晰、具体、可论证

### Conclusion 四阶段结构

1. **重申论点**：用不同表述重述 Thesis Statement
2. **逻辑总结**：简述分论点如何支持核心论点
3. **意义扩展**：
   - 理论贡献：对学术领域的补充/修正
   - 实践价值：对从业者的指导意义
   - 宏观定位：社会价值
4. **局限与展望**：客观说明限制，提出未来方向

**注意**：结论禁止引入新观点、新引用。

## 模块二：翻译

### 中文 → 英文（学术论文）

**角色**：顶尖科研写作专家 + 资深会议审稿人（ICML/ICLR）

**要求**：
- 逻辑严谨，用词准确，表达凝练
- 避免破折号、列表格式，使用连贯段落
- 去除"AI味"，行文自然流畅
- 时态：一般现在时（除特定历史事件）
- LaTeX转义：`95%` → `95\%`，`model_v1` → `model\_v1`

**输出格式**：
- Part 1 [LaTeX]：英文内容（LaTeX格式）
- Part 2 [Translation]：中文直译（核对逻辑）

### 英文 → 中文

**要求**：
- 直译为主，保持原句结构
- 删除 `\cite{}`、`\ref{}`、`\label{}` 等命令
- 数学公式转化为自然语言描述
- 不润色、不增删词汇

## 模块三：润色优化

### 英文润色

**要求**：
- 句式优化：调整结构适配顶级会议规范
- 零错误：修正拼写、语法、冠词错误
- 正式语体：使用 "it is" 而非 "it's"
- 避免名词所有格：用 "the performance of METHOD" 而非 "METHOD's performance"
- 保留 LaTeX 命令和原有格式

### 中文润色

**要求**：
- 仅在口语化、语法错误、逻辑断层时修改
- 原文通顺时严禁强行替换同义词
- 去除口语："我们发现" → "实验结果表明"
- 使用中文全角标点

### 英文缩写

- 目标：减少 5-15 个单词
- 手段：从句变短语、被动变主动、删除填充词（"in order to" → "to"）
- 严禁大删大改，保留核心信息

### 英文扩写

- 目标：增加 5-15 个单词
- 手段：显式化隐含结论、增加必要连接词、升级学术表达
- 严禁恶意注水

## 模块四：AI 痕迹检测与移除

### 高频 AI 词汇（需替换）

| AI词汇 | 替换建议 |
|-------|---------|
| Additionally | 此外/另外/同时 |
| delve into | 研究/探讨/分析 |
| crucial/pivotal/key | 重要的/关键的 |
| underscore/highlight | 表明/显示 |
| leveraging | 利用/使用 |
| testament | 证明/体现 |
| vibrant | 活跃的/丰富的 |
| evolving landscape | 发展领域/现状 |
| in the heart of | 位于/地处 |
| groundbreaking | 创新的/突破性的 |

### 结构模式（需修正）

1. **三段式结构**：将"A、B和C"列表改为连贯段落
2. **否定平行**：避免"Not only...but..."，改用直接陈述
3. **-ing 形式堆砌**：将句末的 highlighting/ensuring/reflecting 改为独立句子
4. **虚假象征**：删除"标志着/象征着/代表了"等夸大表述
5. **破折号滥用**：用逗号或分句替代破折号
6. **模糊归因**：将"Experts argue"改为具体引用

### 内容模式（需修正）

1. **过度强调重要性**：删除"是...的证明/见证了/象征着"
2. **宣传式语言**：删除"令人惊叹的/必去的/绝美的"等形容词
3. **大纲式章节**：删除公式化的"挑战与未来展望"套路
4. **同义词循环**：统一用词，避免"主角/主要角色/核心人物/英雄"轮换

### 改写示例

**Before（AI风格）**：
> 此外，该算法是深度学习领域的重大突破，标志着神经网络发展的新纪元。研究人员对其寄予厚望，认为它将对整个行业产生持久影响。

**After（人工化）**：
> 该算法在ImageNet数据集上将准确率提升了3.2%。实验表明，其在目标检测任务中也表现稳定。

## 模块五：逻辑检查与审稿

### 逻辑检查

**审查维度**：
- 致命逻辑：前后矛盾陈述
- 术语一致性：核心概念是否无故更名
- 严重语病：导致句意不清的表达

**原则**：高容忍度，仅报错实质性问题，忽略"可改可不改"的风格问题。

### Reviewer 审稿

**审查维度**：
- 原创性：实质性突破还是边际增量
- 严谨性：推导是否有跳跃、对比是否公平
- 一致性：引言声称的贡献是否被实验验证

**输出**：
- Summary：一句话总结
- Strengths：1-2点真正有价值的贡献
- Weaknesses：3-5个可能导致拒稿的问题
- Rating：预估评分（1-10分）

## 快速使用指南

### 撰写 Introduction

1. 从宏观背景开始（领域现状）
2. 明确研究必要性（重要性 + 研究空白）
3. 使用 5W1H 界定研究范围
4. 清晰陈述 Thesis Statement
5. 检查：是否回应了"为什么这个研究重要"

### 撰写 Conclusion

1. 用不同表述重申 Thesis Statement
2. 总结论证逻辑（分论点 → 核心论点）
3. 阐述理论贡献与实践价值
4. 提及局限性与未来方向
5. 检查：是否引入了新观点或新引用（应无）

### 降低 AI 率检查清单

- [ ] 是否使用了高频AI词汇（delve, crucial, underscore等）
- [ ] 是否存在三段式列表结构
- [ ] 是否滥用破折号
- [ ] 是否存在模糊归因（Experts argue...）
- [ ] 是否存在过度象征（标志着/象征着）
- [ ] 句子结构是否有变化（长短句交替）
- [ ] 是否包含具体数据和事实

## 撰写顺序建议

1. 先完成正文
2. 再写 Introduction
3. 最后写 Conclusion
4. 全文完成后进行 AI 痕迹检测与润色
5. 终稿前进行逻辑检查
